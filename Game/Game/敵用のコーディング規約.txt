1. 定数定義
1.1 配置と命名
•すべての定数は無名名前空間内に集約すること
•定数名はUPPER_SNAKE_CASEを使用すること
•用途ごとにコメントブロックで分類すること
1.2 係数の定数化
•マジックナンバーの係数は定数として定義すること
•例: 0.8f → WANDER_HOME_RETURN_RATIO
•例: 1.5f → CONFRONT_DISTANCE_RATIO

2. 変数命名規則
2.1 ベクトル変数
•ベクトル変数には接頭辞vを付けること
•方向ベクトル: vDir
•目標への方向: vToTarget, vToHome
•正規化されたベクトル: vToTargetNorm
2.2 距離変数
•距離変数はdistを使用すること
•距離の二乗はdistSqを使用すること
2.3 その他の変数
•速度: speed
•タイマー: _fTimer（メンバ変数）
•パラメータ参照: param または const auto&を使用

3. コメント規約
3.1 処理ブロックコメント
各処理ブロックには以下の形式でコメントを付けること:
•初期化系: 「～初期化」
•判定系: 「～チェック」
•計算系: 「～計算」
•更新系: 「～処理」「～設定」「～更新」
3.2 インラインコメント
•右側コメントは関連する処理の意味を簡潔に記述
•遷移先を明示する場合は「～状態へ」を使用

4. 処理順序の統一
各ステートのUpdateメソッドは以下の順序で実装すること:
1.前提条件チェック (ターゲット存在確認など)
2.タイマー更新
3.ベクトル・距離計算
4.状態遷移判定
5回転処理
6.移動処理
7.その他の更新処理

5. 空行規則
•各処理ブロック間には必ず1行の空行を入れること
•ステート実装間には5行の空行を入れること
•関数定義間には1行の空行を入れること

6. 条件分岐
6.1 早期リターン
条件不成立時は早期リターンを使用すること
6.2 比較演算
•定数を右辺に配置すること
•例: dist > limitRange (推奨)

7. 型推論
autoキーワードは以下の場合に使用すること:
•イテレータ
•複雑な型名の省略
•明らかに型が推測できる場合
使用を避ける場合:
•プリミティブ型(float, intなど)で意図を明確にする必要がある場合

8. DXライブラリ関数の使用
ベクトル演算
ベクトル生成: VECTOR v = VGet(x, y, z);
•減算: VSub(a, b)
•加算: VAdd(a, b)
•正規化: VNorm(v)
•スケール: VScale(v, scale)
•サイズ: VSize(v)
•サイズの二乗: VSquareSize(v)
•外積: VCross(a, b)
